# JSONova

JSONova is a modern JSON workspace for developers. It brings editing, validation, diffing, conversion, JWT inspection, AI assistance, graph exploration, mock generation, and table workflows into one privacy-first product.

## Features

- Monaco-based JSON editor with format, minify, repair, sort, upload, and URL import
- Validation with exact error feedback, stats, search, and sensitive-field detection
- Tree, graph, table, JSONPath, and search workflows for exploring nested payloads
- Converter workspace for TypeScript, Zod, Go, Python, Rust, C#, Java, CSV, YAML, XML, Schema, Prisma, and Mongoose
- JSON diff workspace with side-by-side comparison and readable summary cards
- JWT decoder with expiry details and HS256 verification
- AI assistant powered by Gemini for explain, fix, query, and generate tasks
- Mock data generator and persistent local history

## Getting started

```bash
npm install
cp .env.example .env.local
```

Add your Gemini key to `.env.local`:

```bash
GEMINI_API_KEY=your_gemini_api_key_here
```

Run the app:

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Environment variables

| Variable                       | Required   | Purpose                                    |
| ------------------------------ | ---------- | ------------------------------------------ |
| `GEMINI_API_KEY`               | Yes for AI | Gemini API key for the AI assistant        |
| `GEMINI_MODEL`                 | No         | Defaults to `gemini-2.5-flash-lite`        |
| `NEXT_PUBLIC_APP_NAME`         | No         | Public app label                           |
| `NEXT_PUBLIC_APP_URL`          | No         | Canonical app URL                          |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | No         | Enables Plausible analytics                |
| `UPSTASH_REDIS_REST_URL`       | No         | Shared rate limiter backend for production |
| `UPSTASH_REDIS_REST_TOKEN`     | No         | Shared rate limiter backend token          |

## Project structure

```text
src/
  app/                 Routes, metadata, API handlers
  components/          Shared app-level components
  components/ui/       UI primitives
  features/home/       Homepage sections
  features/workspace/  Product workspace modules
  hooks/               Reusable client hooks
  lib/                 Utilities, prompts, SEO helpers
  workers/             Background parsing workers
```

## Deployment notes

- Core JSON processing is local-first
- AI requests leave the browser only through `/api/ai`
- In production, set Upstash Redis variables for shared AI rate limiting
- `robots.txt`, `sitemap.xml`, and `/opengraph-image` are generated by the app
